# Learner Profile Template

Use this template to create a new learner profile. Save completed profiles to
`socratic-learner-profiles/[student-identifier].md` in the workspace.

---

```markdown
# Learner Profile: [Student Name or Identifier]

## Summary (updated each session)
- Primary discipline: [X]
- Current language calibration: Level [1/2/3]
- Current provocation intensity: [Introductory / Developing / Rigorous / Adversarial]
- Sessions completed: [N]
- Last session date: [date]

## Active Reasoning Patterns

### Recurring Strengths
[List the reasoning behaviours this student consistently demonstrates well.
Be specific: "strong evidentiary reasoning" is better than "good analysis."]

### Recurring Weaknesses
[List the reasoning gaps that recur across sessions. Be specific: "tends to
overstate conclusions beyond what the evidence supports" is better than
"weak arguments."]

### Blind Spots
[Lenses or challenge types this student consistently fails to engage with,
even when clearly relevant. E.g., "rarely considers jurisdictional specificity
even when the analysis is jurisdiction-dependent."]

## Growth Notes
[Brief narrative: what has improved since the earliest sessions? What remains
persistent? Are new challenges emerging as the student tackles harder material?]

## Calibration History
[Track changes to language level and provocation intensity, with the reason
for each change.]

- [Date]: Language [X], Intensity [X] — initial assessment
- [Date]: Intensity raised to [X] — student demonstrated generative engagement
  for 3 consecutive sessions, previously identified weaknesses (overstating
  claims, ignoring scope) are being addressed
- [Date]: Language raised to [X] — student's written responses showing increased
  clause density and rhetorical sophistication

---

## Session Log

### Session 1: [Date] — [Topic]
- Discipline: [X]
- Language: Level [X] | Intensity: [X]
- Provocations used: [list of lenses]
- Engagement quality: [dismissive / surface / substantive / generative]
- Key observations:
  - [What stood out about this student's reasoning in this session]
  - [Specific strengths demonstrated]
  - [Specific weaknesses demonstrated]
  - [How the student handled the most difficult provocation]
- Recommendation for next session: [any adjustments to calibration, lenses
  to prioritise, patterns to probe]

### Session 2: [Date] — [Topic]
[Same structure. Include comparison to Session 1 where relevant.]

[Continue for each session.]
```

---

## Notes on Profile Maintenance

**Identifiers:** Use whatever the student provides — their name, a pseudonym, a student
ID. If the student does not provide an identifier, use the topic and date as a session
label and ask whether they would like a persistent profile.

**Privacy:** Learner profiles contain observations about a student's reasoning patterns.
They should be treated as educational records. In an instructor-mediated context, the
instructor decides whether profiles are shared with students (recommended — transparency
builds trust). In a student-direct context, the student owns their profile.

**When a student works across disciplines:** A student who uses this skill for both
constitutional law and philosophy may show different calibration levels in each. The
profile should note discipline-specific calibration rather than assuming a single level
applies everywhere. A student who is analytically rigorous in philosophy but a novice in
empirical methods should not be given Intensity 4 provocations in a statistics context
just because they handle them well in philosophy.

**When to create a new profile vs. update an existing one:** If the student is clearly
the same person returning for another session, update the existing profile. If in doubt,
ask. Do not create duplicate profiles.
